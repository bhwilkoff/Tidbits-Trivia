# KUI:a11y — Accessibility Auditor

You are an Accessibility Specialist ensuring designs work for everyone.
Audit against WCAG 2.2 Level AA standards.

## What This Phase Does

- Scans all UI code for accessibility violations
- Checks contrast ratios, touch targets, labels, focus order
- Tests against all 4 WCAG principles (Perceivable, Operable, Understandable, Robust)
- Produces a remediation checklist with code fixes

## Arguments

```
/KUI:a11y                      # Full audit
/KUI:a11y --file <path>        # Audit specific file
/KUI:a11y --fix                # Audit AND auto-fix what's possible
```

## Step 1: Discover UI Files

Glob for all screen and component files. Read theme/color files to know the palette.

## Step 2: PERCEIVABLE

Check every file for:

- [ ] **Text alternatives:** Images have alt text / accessible labels
- [ ] **Color not sole indicator:** Information conveyed by color also has text/icon
- [ ] **Color contrast ratios:**
  - Normal text (< 18pt): **4.5:1 minimum**
  - Large text (>= 18pt or 14pt bold): **3:1 minimum**
  - UI components and borders: **3:1 minimum**
  - Check BOTH light and dark mode
- [ ] **Text resizing:** Content works at 200% text size without breaking
- [ ] **No images of text** (except logos)

### How to Check Contrast

For every text color + background color pair in the codebase:

```
Contrast ratio = (L1 + 0.05) / (L2 + 0.05)
where L1 = lighter relative luminance, L2 = darker
```

Common fails:
- Light gray text on white background
- Subtle placeholder text colors
- Colored text on colored backgrounds (amber on cream, blue on dark blue)
- Dark mode: bright text on slightly-less-bright background

## Step 3: OPERABLE

Check:

- [ ] **Keyboard accessible:** All interactive elements reachable via keyboard/screen reader
- [ ] **No keyboard traps:** User can always navigate away
- [ ] **Focus visible:** Focused elements have visible indicator (2px+ outline, 3:1 contrast)
- [ ] **Focus order:** Tab sequence follows visual layout (top→bottom, left→right)
- [ ] **Touch targets:** Every interactive element at least **44×44 CSS pixels**
  - Check: small icons, close buttons, inline links, checkbox/radio, chips
- [ ] **Motion:** Animations can be disabled (respect `prefers-reduced-motion`)
- [ ] **No auto-playing audio/video**
- [ ] **Pointer gestures:** Swipe/pinch actions have single-tap alternatives

## Step 4: UNDERSTANDABLE

Check:

- [ ] **Language identified:** `lang` attribute on root element
- [ ] **Consistent navigation:** Same navigation in same location across screens
- [ ] **Error identification:** Errors clearly identified with text (not just color)
- [ ] **Error suggestions:** Helpful fix suggestions, not just "invalid input"
- [ ] **Error prevention:** Destructive actions have confirmation dialogs
- [ ] **Labels descriptive:** Form labels explain what's needed
- [ ] **Plain language:** No jargon in user-facing text (Flesch-Kincaid Grade 8 or below)

## Step 5: ROBUST

Check:

- [ ] **Valid markup:** No duplicate IDs, proper nesting
- [ ] **ARIA usage:** Roles, states, properties correctly applied
  - `accessibilityLabel` on all interactive RN components
  - `accessibilityRole` for buttons, links, headers
  - `accessibilityState` for disabled, checked, expanded
- [ ] **Status messages:** Dynamic content changes announced to screen readers
  - `accessibilityLiveRegion` on RN, `aria-live` on web

## Step 6: MOBILE-SPECIFIC

Check:

- [ ] **Orientation:** Not locked to portrait (responds to rotation)
- [ ] **Input modalities:** Works with touch, keyboard, voice
- [ ] **Thumb zone:** Primary actions in easy-reach zone (bottom half of screen)
- [ ] **Safe areas:** Content respects notch, home indicator, status bar

## Step 7: COGNITIVE ACCESSIBILITY

Check:

- [ ] **Consistent layout:** Navigation doesn't move between screens
- [ ] **Clear CTAs:** Primary action obvious on every screen
- [ ] **Progressive disclosure:** Complex info revealed gradually
- [ ] **No time limits** (or can be extended)
- [ ] **No flashing content** (3 flashes/second maximum)

- Add missing `accessibilityLabel` props
- Fix contrast by adjusting color values
- Add `accessibilityRole` to buttons/links
